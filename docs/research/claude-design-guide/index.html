Start Here

Quick Start

The home screen is built around a prompt composer and project starters. Pick the output type and generation settings first, then send a clear design request.

  • Open https://claude.ai/design.
  • Type the outcome under "What will you design today?"
  • Choose a design system when the output should follow a saved style.
  • Choose a template when you want a reusable starting structure.
  • Pick the model and effort level before sending.
  • Use the starter cards for Prototype, Slides, Document, Wireframe, or Animation.
  • Use the blank project link when you want an empty workspace first.

Home Screen

Prompt Composer And Starters

  • The large text box is the main entry point for a new Claude Design project.
  • The placeholder examples show the range of supported outputs.
  • The plus control adds input or context before generation.
  • The selector row controls design system, template, model, and effort.
  • The send button starts generation after the prompt and options are set.

Generation Controls

Design System, Template, And Model Pickers

These controls decide how Claude should shape the first draft before generation begins.

Design system picker

  • Bind a project to a saved visual system.
  • Search existing design systems from the picker.
  • Create a new design system from the picker when needed.
  • Use this when consistency matters more than open-ended exploration.

Template picker

  • Pick a project type before generation: Slides, Prototype, Wireframe, Document, or Animation.
  • Search saved templates when the workspace has reusable templates.
  • Create templates from existing projects by duplicating a project as a template.

Model picker

  • Choose the Claude model used for the design task.
  • Use higher-capability models for complex generation and faster models for quick iterations.
  • Adjust effort when the task needs deeper reasoning or a lighter pass.
  • Disabled models can appear when they are unavailable in the current account state.

Management Views

Projects, Design Systems, And Templates

  • Projects lists existing Claude Design projects with search, star, sorting, owner, and row actions.
  • Design systems lists saved systems and exposes the create-design-system action.
  • Templates lists saved reusable project templates and shows empty-state guidance when none exist.
  • Open any project, design system, or template from its row in the library.

Project Workspace

Iterate, Preview, Navigate, And Share

  • The left pane keeps the conversation, generated responses, screenshots, edits, and status messages.
  • The bottom composer is where users request changes and attach context.
  • The composer keeps the active design system, project chip, model, effort, and send controls visible.
  • The right pane previews the generated page or file.
  • The preview toolbar includes reload, page selector, markup, comments, edit, tweaks, zoom, present, and share.
  • Usage-limit, report, and copy-debug-info states are visible when a session cannot continue generation.

Project controls and chat history

  • Use the project menu for project-level actions such as copying debug info or reporting a problem.
  • Use chat history to move between prior conversations without leaving the project.
  • Keep the conversation pane open when you need to trace what Claude changed and why.

Page and file navigation

  • Use the page selector to switch between generated pages.
  • Create a new blank page from the selector.
  • Duplicate or rename existing pages from their page actions.
  • Open all project files when you need to inspect files beyond the main pages.

Preview, zoom, and share controls

  • Reload the preview when the generated page needs to refresh.
  • Use zoom controls to inspect the design at a comfortable scale.
  • Use share only after reviewing the project and confirming it is ready for others.

Recommended Flow

Practical Workflow

01 Frame the output

Choose prototype, slides, document, wireframe, animation, or blank project.

02 Set constraints

Pick a design system, template, model, and effort level before generation.

03 Generate the draft

04 Review the preview

Use the right pane to inspect layout, content, and page state.

Use present or share only when the design is ready for others.

1. Artifacts And Visual Output

Use these when a user wants Claude to create something visible, reusable, or shareable.

  • What are artifacts and how do I use them?
    • Ask Claude for a substantial output such as an app, chart, document, tool, or visualization.
    • Open the artifact panel to review the generated output beside the conversation.
    • Iterate by asking Claude to edit the artifact instead of starting over.
  • Publish and share artifacts
    • Review the artifact first.
    • Use publish or share only when the content is ready for others.
    • Re-check sensitive text, files, and generated code before sharing externally.
  • Can Claude produce images?
    • Ask Claude for visual help when you need interpretation, layout ideas, or image-related reasoning.
    • Verify whether the current Claude surface supports image generation or only image understanding.
  • How does Claude handle mathematical equations and calculations?
    • Ask for equations in readable math notation.
    • Ask Claude to show steps when accuracy matters.
    • Check important calculations independently.
  • Visual and interactive content
    • Use Claude for prototypes, diagrams, interactive explainers, and visual artifacts.
    • Iterate from the preview with concrete change requests.
  • Custom visuals in chat and Cowork
    • Ask for visuals directly in the relevant chat or Cowork flow.
    • Use the visual result as a draft unless brand or compliance review is required.

2. Search, Research, And Files

Use these when Claude needs outside information, uploaded context, or generated files.

  • When should I use web search, extended thinking, and research?
    • Use web search for current facts.
    • Use extended thinking for harder reasoning.
    • Use research for broader investigation across sources.
  • Enable and use web search
    • Turn on web search where available.
    • Ask time-sensitive questions with clear dates, locations, or constraints.
    • Read citations before acting on high-stakes answers.
  • Use research on Claude
    • Start with a research question and the outcome format you need.
    • Let Claude gather sources, then ask it to compare, summarize, or turn findings into a deliverable.
  • Upload files to Claude
    • Attach the file.
    • Tell Claude what to extract, compare, summarize, or transform.
    • Avoid uploading secrets or private files unless the workspace policy allows it.
  • Create and edit files with Claude
    • Ask Claude to create the target file type.
    • Review the generated file.
    • Ask for edits in plain language instead of manually rebuilding the file.

3. Projects, Knowledge, Memory, And Skills

Use these when work needs persistent context or reusable instructions.

  • What are projects?
    • Create a project for work that shares files, instructions, and ongoing context.
    • Add project knowledge before asking project-specific questions.
    • Keep unrelated work in separate projects.
  • Retrieval augmented generation (RAG) for projects
    • Add source documents to the project.
    • Ask questions that should be grounded in those documents.
    • Check citations or source references before relying on the answer.
  • Import and export your memory from Claude
    • Export memory when moving or backing up context.
    • Import memory when restoring context in a supported Claude environment.
  • What are skills?
    • Use skills when Claude should follow repeatable instructions or workflows.
    • Prefer skills for recurring tasks instead of re-prompting from scratch.
  • Use skills in Claude
    • Browse or select the skill.
    • Give the task.
    • Let the skill shape Claude's process and output.
  • How to create custom skills
    • Write the reusable task instructions.
    • Add examples or supporting files only when they are needed.
    • Test the skill on one real task before sharing it.

4. Connectors, Plugins, And Directories

Use these when Claude needs another service, tool, or managed integration.

  • Browse skills, connectors, and plugins in one directory
    • Open the directory.
    • Search for the tool or category.
    • Review permissions before installing or enabling anything.
  • Use plugins in Claude
    • Install or enable the plugin.
    • Authorize the external service if required.
    • Ask Claude to use the plugin for a specific task.
  • Install financial services plugins
    • Confirm account and workspace eligibility.
    • Review data access carefully.
    • Use the plugin only for the financial workflow it is meant to support.

5. Office, Developer, And Platform Integrations

Use these when Claude runs inside another app instead of the main Claude chat.

  • Use Claude for Excel
    • Open Claude inside Excel.
    • Ask for spreadsheet analysis, formula help, cleanup, or summaries.
    • Review formulas and transformed data before saving.
  • Use Claude for PowerPoint
    • Ask Claude to draft, rewrite, or improve slides.
    • Review slide structure, tone, and speaker context.
  • Use Claude for Word
    • Ask Claude to draft, summarize, rewrite, or review document content.
    • Check tracked changes or inserted text before sharing.
  • Use Claude for Outlook
    • Ask Claude to draft, summarize, or improve email.
    • Verify recipients, attachments, and tone before sending.
  • Work across Microsoft 365 apps
    • Use Claude when the task spans documents, slides, spreadsheets, or email.
    • Name the source app and target output clearly.
  • Use Claude for Microsoft 365 with third-party platforms
    • Confirm the platform integration is installed.
    • Check what Microsoft 365 data the third-party platform can access.
  • Use dictation in Office agents
    • Turn on dictation where available.
    • Speak the request or content.
    • Review transcribed text before applying or sending.
  • Use Claude in Xcode
    • Use Claude for coding help inside the Xcode workflow.
    • Review generated code and run the app or tests locally.
  • Use Claude in Microsoft Foundry
    • Configure Claude in Foundry.
    • Use it through the Foundry workflow and deployment controls.
    • Validate behavior against the target environment.

6. Claude Design

Use these when a user wants generated prototypes, documents, slides, wireframes, animations, or design-system-bound output.

  • Get started with Claude Design
    • Open Claude Design.
    • Choose a design system, template, model, and output type.

7. Beta, Preview, And Security Features

Use these when availability, risk, or workspace controls matter.
